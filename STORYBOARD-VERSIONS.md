## How to Update Versions

When making changes to any storyboard:

1. Determine change type (MAJOR/MINOR/PATCH)
2. Update version in this file (STORYBOARD-VERSIONS.md)
3. Add changelog entry describing changes
4. Update version header in ALL storyboard files (even unchanged ones)

**Version header format in storyboards:**
```markdown
**Version:** 1.0.0 | **Last Updated:** 2025-01-27
```

## Files Excluded from Versioning

The following are not versioned storyboards:
- `README.md` files (navigation/documentation)
- `*-rev.md`, `*-old.md` files (archives/backups)
- `PAIRR-*.md`, `*-PAIRR-*.md` files (working documents)
- `.ORIGINAL` files (backup copies)
